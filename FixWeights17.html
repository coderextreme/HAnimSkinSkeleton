X3D Rigging Fixer: Hierarchy Region Enforcer v17

Reads the actual HAnim joint hierarchy (parent→child XML nesting) to classify every joint as ARM (elbow and below) or LEG (hip and below). Any vertex influenced by joints from both regions has the minority-weight region stripped — no raycasting, no air-gap tolerance to tune.

Everything from these joints downward in the hierarchy is treated as "arm". Use l_shoulder, r_shoulder to include shoulders. Default starts at the elbow. Everything from these joints downward (knee, ankle, toes…) is "leg". If the original file had vertices assigned only to arm joints but they sit below the waist joint (vl5 / sacrum / pelvis), strip those arm weights too. Catches badly-bled rigs where even the minority test can't fire because no leg weight survived.
